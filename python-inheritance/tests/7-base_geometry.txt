Using ``integer_validator()``
-------------------------------

Valid values return ``None`` and raise nothing:

    >>> bg.integer_validator("my_int", 12)
    >>> bg.integer_validator("width", 89)

A non-integer value raises a ``TypeError``:

    >>> bg.integer_validator("name", "John")
    Traceback (most recent call last):
    TypeError: name must be an integer

Calling ``integer_validator()`` with no arguments at all raises a
``TypeError`` for the missing required arguments:

    >>> bg.integer_validator()
    Traceback (most recent call last):
    TypeError: BaseGeometry.integer_validator() missing 2 required positional arguments: 'name' and 'value'

Using ``integer_validator("age")``
-------------------------------------

Calling ``integer_validator("age")`` with only the ``name`` argument
and no ``value`` also raises a ``TypeError``:

    >>> bg.integer_validator("age")
    Traceback (most recent call last):
    TypeError: BaseGeometry.integer_validator() missing 1 required positional argument: 'value'

A value less than or equal to 0, using the name ``"age"``, raises a
``ValueError``:

    >>> bg.integer_validator("age", 0)
    Traceback (most recent call last):
    ValueError: age must be greater than 0

    >>> bg.integer_validator("age", -12)
    Traceback (most recent call last):

types or ``bool`` (since ``bool`` is not exactly ``int``):

    Traceback (most recent call last):
