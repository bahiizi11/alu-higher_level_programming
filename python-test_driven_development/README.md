# python-test_driven_development

This project introduces Test-Driven Development in Python: writing
functions alongside doctests (interactive tests embedded in `.txt`
files under `tests/`) and standard `unittest` test cases, so that
implementation and tests are developed together.

## Learning Objectives

- What's a doctest and how to write one
- What's unittest and how to use it
- How to write TDD before/alongside implementation
- Why testing is important in large projects

## Files

| File | Description |
|------|-------------|
| `0-add_integer.py` / `tests/0-add_integer.txt` | Adds two integers |
| `2-matrix_divided.py` / `tests/2-matrix_divided.txt` | Divides all elements of a matrix |
| `3-say_my_name.py` / `tests/3-say_my_name.txt` | Prints a person's full name |
| `4-print_square.py` / `tests/4-print_square.txt` | Prints a square of `#` |
| `5-text_indentation.py` / `tests/5-text_indentation.txt` | Formats text with indentation |
| `6-max_integer.py` / `tests/6-max_integer_test.py` | Finds the max integer in a list (unittest) |
