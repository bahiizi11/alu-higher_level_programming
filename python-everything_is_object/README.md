# python-everything_is_object

This project explores one of Python's core principles: everything is an
object. It covers object identity vs. equality, mutable vs. immutable
types, and how arguments are actually passed to functions in Python.

## Learning Objectives

- Why Python programming is awesome
- What's the official Python coding style and how to check your code with pycodestyle
- Everything in Python is an object
- What's the difference between a class and an object or instance
- What's the difference between `id(a) == id(b)` and `a == b`
- What are mutable and immutable objects
- What's a reference and how do you assign a reference to a variable
- What's an alias for a reference and how to know if two variables are identical
- How to know if two variables are linked to the same object (`is` vs `==`)
- How does Python pass variables to functions
