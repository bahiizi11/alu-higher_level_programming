# python-network_0

This project covers the fundamentals of HTTP and networking from the
command line using `curl` in Bash scripts: inspecting response sizes,
bodies, and headers, sending different HTTP methods (GET, DELETE,
OPTIONS, POST), and passing custom headers and POST parameters.

## Learning Objectives

- What's a URL, and the HTTP request/response cycle
- What's `curl` and how to use it to make requests from the CLI
- How to send GET, DELETE, and POST requests with `curl`
- How to send custom headers with a request
- How to check the HTTP status code and body size of a response
- What HTTP methods are, and how to discover which ones a server accepts

## Files

| File | Description |
|------|-------------|
| `0-body_size.sh` | Displays the size in bytes of a response body |
| `1-body.sh` | Displays the response body only on a `200` status |
| `2-delete.sh` | Sends a `DELETE` request and displays the body |
| `3-methods.sh` | Displays all HTTP methods a server accepts |
| `4-header.sh` | Sends a `GET` with a custom header, displays the body |
| `5-post_params.sh` | Sends a `POST` with `email`/`subject` params |
